# X（Twitter）投稿スケジュール

delilog ブログ記事を X で再投稿（リサイクル）するための運用管理ファイル。

- 軽貨物ドライバーの生活リズムに合わせた **時間帯固定** ＋ **角度を変えたリサイクル投稿** で運用する
- 投稿実績は `utm_content` で識別し、Google Analytics で効果検証する
- 公開記事は `docs/BLOG_BACKLOG.md` を参照

最終更新: 2026-05-03（記事 #9 公開・マトリクス追加）

---

## 1. 基本ルール

### 投稿時間帯（固定3枠）

軽貨物ドライバーが SNS を見る生活リズムに合わせ、以下の3枠を固定で運用する。

| 枠 | 曜日 | 時刻 | 想定シーン | 適したコンテンツ |
|----|------|------|------------|------------------|
| **早朝** | 月〜土 | 5:45 | 配送出発前のスマホチェック | 結論・チェックリスト・要点 |
| **夜** | 月〜金 | 21:00 | 帰宅後・風呂上がり | 解説スレッド・読み物 |
| **日曜午前** | 日 | 10:00 | 週1の休日、情報収集モード | 長尺記事の本格紹介 |

**避ける時間帯**:
- 14:00〜17:00（配送ピーク・スマホに触れない）
- 金曜夜（飲食店配送ピーク）
- 月初・月末の月曜朝（請求書処理で集中力低い）

### リサイクル運用のルール

1. **同じ記事は最低3週間あけて再投稿**
2. 角度（切り口）を変えて4パターンで回す
3. UTM の `utm_content` で投稿パターンを識別する
4. 投稿実績はこのファイルに記録する

### 角度（切り口）テンプレ

| 角度 | utm_content | 文体パターン | 使う時間帯 |
|------|-------------|-------------|-----------|
| **結論型** | `initial` | 「〇〇は△△で完結します」 | 早朝 |
| **痛み・失敗型** | `fail-case` | 「『□□しなきゃ』で消耗してませんか?」 | 夜 |
| **FAQ抜粋** | `faq` | 「『〜は必須?』→ 答えはNo」 | 早朝 |
| **チェックリスト/図表** | `checklist` | 記事中の表・図を画像で抜粋 | 日曜午前 |

## 2. UTM 設計

```
https://delilog.app/blog/<slug>?utm_source=x&utm_medium=social&utm_content=<角度>
```

例:
```
https://delilog.app/blog/tenko-kiroku-kakikata?utm_source=x&utm_medium=social&utm_content=fail-case
```

- 詳細ルールは `docs/UTM_GUIDE.md` 参照
- すべて小文字、`utm_content` の値は固定 4種（`initial` / `fail-case` / `faq` / `checklist`）

## 6. X で予約投稿する方法

### 推奨: X 公式の予約投稿機能（無料）

**Web版（x.com）の投稿画面で予約可能**。モバイルアプリでは設定できないので、PC または ブラウザのモバイル版を使う。

#### 手順

1. [x.com](https://x.com) にログイン
2. 「ポストする」ボタンを押して投稿画面を開く
3. 投稿画面下のアイコン群の中の **カレンダーアイコン** をクリック
4. 日時を指定して「確認」→「予約設定」をクリック
5. 予約済み投稿は [https://x.com/compose/post/unsent/scheduled](https://x.com/compose/post/unsent/scheduled) で確認・編集可能

> 公式機能なのでアカウントBANリスクなし。**無料**で件数制限なく予約可能。

### 代替手段

| ツール | 料金 | 特徴 |
|--------|------|------|
| **SocialDog** | 無料10件 / Lite ¥980〜 | 日本産、UI日本語、複数SNS対応 |
| **Buffer** | 無料10件 / Pro $6〜 | 海外定番、複数SNS、英語UI |
| **Typefully** | 無料 / Pro $12.5〜 | スレッド作成に特化、予約可 |
| **Hootsuite** | 無料廃止 / $99〜 | 高機能だが個人には過剰 |

#### SocialDog がおすすめなケース
- 1日3枠×複数記事を一気に1週間分予約したい
- カレンダーUIで「いつ何を投稿予定か」を俯瞰したい
- 投稿パフォーマンスをツール上で分析したい

#### Typefully がおすすめなケース
- スレッド形式（連投）を頻繁に使う
- 下書きをチームで共同編集したい

### 当面の運用

**まずは X 公式の予約投稿で十分**。週末（日曜夜）にまとめて1週間分（18件）を予約しておけば、平日は手を動かさず運用できる。

慣れてきて「カレンダーUIで管理したい」「分析がしたい」と感じたら SocialDog の有料プランに切り替えを検討。

## 7. 月次レビュー（毎月1日）

### チェック項目

1. **GA で `utm_content` 別の流入確認**
   - GA → 集客 → トラフィック獲得 → セッションのデフォルトチャネルグループ
   - フィルタで `utm_source = x` & `utm_content = <値>` を比較
2. **時間帯別の反応**
   - X アナリティクス → ポスト → インプレッション・エンゲージメント
3. **どの角度が刺さったかを記録**
   - 上位3パターンを翌月の投稿配分に反映
4. **このファイルの「公開済み記事 × 角度マトリクス」を更新**

### 改善ポイント

- 反応の悪い時間帯は別の時間帯に振り替え
- 反応の良い角度を増やす（例: 痛み型が強ければ週の半分を痛み型に）
- 8記事を1周したら、上位記事を**Pinned Tweet** に固定
